*Precondiciones:*

El nombre de usuario y contraseña no deben ser nulos ni vacíos.

El sistema debe estar conectado a la base de datos (ConexionDB activa).

*Postcondiciones:*

Si las credenciales son válidas, retorna true y carga el objeto Usuario asociado.

Si las credenciales son incorrectas, retorna false sin alterar el estado del sistema.

El sistema guarda los cambios dentro de la base de datos referentes a usuarios nuevos, eliminados o actualizados.

===== Interfaz de Pago

image::IPagoConTarjeta.png[Diagrama de Especificación de interfaces de Interfaz Pago, width=400, align=center]

*Precondiciones:*

El usuario debe tener una reserva confirmada.

Los datos de pago deben ser válidos (método, monto, referencia).

*Postcondiciones:*

Si el pago se procesa correctamente, se genera un registro en la base de datos y se actualiza el estado de la reserva.

Si el pago falla, no se generan registros ni cambios de estado.

===== Interfaz de Reserva

image::IAgregarServicioExtra.png[Diagrama de Especificación de interfaces de Interfaz Reserva, width=400, align=center]

image::ICheckIn.png[Diagrama de Especificación de interfaces de Interfaz Reserva, width=400, align=center]

image::ICheckOut.png[Diagrama de Especificación de interfaces de Interfaz Reserva, width=400, align=center]

image::IConsultarDisponibilidad.png[Diagrama de Especificación de interfaces de Interfaz Reserva, width=300, align=center]

image::IConsultarReserva.png[Diagrama de Especificación de interfaces de Interfaz Reserva, width=300, align=center]

image::ICancelarReserva.png[Diagrama de Especificación de interfaces de Interfaz Reserva, width=300, align=center]

image::IRegistrarReserva.png[Diagrama de Especificación de interfaces de Interfaz Reserva, width=300, align=center]

*Precondiciones:*

La habitación debe estar disponible en las fechas solicitadas.

El usuario debe estar autenticado en el sistema.

*Postcondiciones:*

Si la reserva es válida, se almacena en la base de datos y se actualiza el estado de la habitación.

Si se cancela, la habitación vuelve a estado “disponible” y la reserva pasa a estado “cancelada”.
