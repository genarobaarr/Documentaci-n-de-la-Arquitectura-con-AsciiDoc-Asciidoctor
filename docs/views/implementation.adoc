==== Diagramas de especificación de interfaces

===== Interfaz de Habitación

image::interfaceSpecHabitacion.png[Diagrama de Especificación de interfaces de Interfaz Habitación, width=300, align=center]

*Precondiciones:*

Las fechas deben ser válidas (fechaInicio ≤ fechaFin).

La habitación debe existir en el registro del sistema.

*Postcondiciones:*

Si se actualiza el estado, este cambio se refleja en la base de datos.

Las consultas retornan listas no nulas (pueden ser vacías si no hay resultados).

===== Interfaz de Login

image::interfaceSpecLogin.png[Diagrama de Especificación de interfaces de Interfaz Login, width=300, align=center]

*Precondiciones:*

El nombre de usuario y contraseña no deben ser nulos ni vacíos.

El sistema debe estar conectado a la base de datos (ConexionDB activa).

*Postcondiciones:*

Si las credenciales son válidas, retorna true y carga el objeto Usuario asociado.

Si las credenciales son incorrectas, retorna false sin alterar el estado del sistema.

===== Interfaz de Pago

image::interfaceSpecPago.png[Diagrama de Especificación de interfaces de Interfaz Pago, width=400, align=center]

*Precondiciones:*

El usuario debe tener una reserva confirmada.

Los datos de pago deben ser válidos (método, monto, referencia).

*Postcondiciones:*

Si el pago se procesa correctamente, se genera un registro en la base de datos y se actualiza el estado de la reserva.

Si el pago falla, no se generan registros ni cambios de estado.

===== Interfaz de Reporte

image::interfaceSpecReporte.png[Diagrama de Especificación de interfaces de Interfaz Reporte, width=500, align=center]

*Precondiciones:*

El usuario solicitante debe tener rol de administrador o auditor.

Las fechas ingresadas deben ser válidas y pertenecer a un rango cronológico coherente.

*Postcondiciones:*

El sistema devuelve una lista de reportes generados dinámicamente con base en los datos actuales.

Los reportes se guardan o actualizan en la tabla de historial (ReporteDAO).

===== Interfaz de Reserva

image::interfaceSpecReserva.png[Diagrama de Especificación de interfaces de Interfaz Reserva, width=400, align=center]

*Precondiciones:*

La habitación debe estar disponible en las fechas solicitadas.

El usuario debe estar autenticado en el sistema.

*Postcondiciones:*

Si la reserva es válida, se almacena en la base de datos y se actualiza el estado de la habitación.

Si se cancela, la habitación vuelve a estado “disponible” y la reserva pasa a estado “cancelada”.
